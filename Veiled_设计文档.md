# Veiled — 隐身AI助手客户端 设计文档

> 项目代号：Veiled（面纱）
> 定位：一个从安装到运行全程伪装为系统组件的本地AI对话客户端

---

## 1. 核心设计原则

- **隐形优先**：所有UI元素默认不可见，仅通过用户主动操作唤出
- **伪装一致**：从文件名、进程名、安装路径到通知内容，全套伪装为系统显示驱动辅助工具
- **即装即用**：安装后无需重启系统，自动启动后台服务
- **回答简洁**：AI输出以"用户能直接用"为标准，不输出冗长解释

## 2. 伪装体系

### 2.1 身份伪装

| 项目 | 伪装内容 |
|------|---------|
| 产品名称 | Windows Display Adapter Helper |
| 安装包文件名 | `DisplayAdapterHelper_Setup.exe` |
| 安装路径 | `C:\Program Files\Windows Display Adapter Helper\` |
| 主程序文件名 | `DAHService.exe` |
| 进程描述 | Microsoft Display Adapter Helper Service |
| 计划任务名 | `\Microsoft\Windows\DisplayAdapter\DAHUpdate` |
| 配置文件 | `adapter_config.dat`（加密） |
| 本地数据库 | `cache.db`（加密存储对话记录） |
| 系统托盘hover文字 | Display Adapter Helper |
| 设置窗口标题 | Display Adapter Configuration |

> 以上命名仅为参考方向。实现时可自行调整，保持"看起来像系统/驱动组件"的风格即可。

### 2.2 文件结构伪装

安装目录下放置若干伪装文件，使目录看起来像驱动包：

```
C:\Program Files\Windows Display Adapter Helper\
├── DAHService.exe          ← 主程序
├── adapter_config.dat      ← 加密配置（API Key、快捷键等）
├── cache.db                ← 加密对话记录
├── display_adapter.inf     ← 伪装文件（空/假内容）
├── dahdriver.sys           ← 伪装文件（空/假内容）
└── README.txt              ← 伪装文件，写一些驱动说明
```

### 2.3 进程伪装

- 打包时设置PE文件的VersionInfo，使其在任务管理器"详细信息"中显示为系统服务风格
- 空闲内存占用控制在 50MB 以内
- 网络请求的User-Agent伪装为普通浏览器

## 3. 安装流程

安装阶段**不需要隐藏**，但界面风格伪装为系统驱动安装向导。

### 3.1 安装向导UI

- 风格：模仿Windows驱动安装向导，灰色朴素界面
- 窗口标题："Windows Display Adapter Helper - Setup"
- 进度条文字："正在安装驱动程序组件..."
- 安装完成后**不弹"安装成功"弹窗**，直接进入首次初始化流程

### 3.2 安装行为

1. 释放文件到安装目录
2. 创建计划任务（用户登录时自启动）
3. 可选：写注册表Run键作为备用自启
4. 自动启动主程序，进入首次初始化

## 4. 首次初始化流程

安装完毕后自动弹出初始化向导。这是**唯一一次**主动显示正式窗口的场景。

### Step 1：API配置

- 选择AI服务商（下拉菜单/选项卡）
  - 预置选项：Claude / OpenAI / 自定义endpoint
  - 填入API Key（密码框样式，可切换显示）
  - "测试连接"按钮，验证Key有效性
- 支持后续在设置中修改和添加多个服务商配置

### Step 2：快捷键绑定

用户**直接按下键盘组合键**来录入，不从列表选择。需要绑定：

| 功能 | 是否必填 | 默认建议 |
|------|---------|---------|
| 唤起/隐藏对话窗 | 必填 | `Ctrl+Shift+Space` |
| 紧急隐藏一切（老板键） | 必填 | `` Ctrl+` `` |
| 剪贴板快捷问答 | 可选 | `Ctrl+Shift+Q` |
| 截图发送AI | 可选 | `Ctrl+Shift+S` |
| 退出程序 | 必填 | `Ctrl+Shift+Alt+Q` |

- 每个热键实时做冲突检测（与系统热键、常见软件热键对比）
- 冲突时给出提示，但不强制阻止

### Step 3：显示偏好

| 设置项 | 类型 | 默认值 |
|--------|------|--------|
| 系统托盘图标 | 开关 | 关 |
| 对话窗弹出位置 | 选择 | 屏幕右下角 |
| 对话窗透明度 | 滑条 50%-100% | 90% |
| 截屏保护 | 开关 | 开 |
| 开机自启动 | 开关 | 开 |
| 回答通知伪装来源 | 选择 | QQ / 微信 / 浏览器 / 无伪装 |

### Step 4：完成

- 显示用户设置的核心热键（大字醒目）
- 明确提示："点击完成后程序进入后台，本窗口不再出现"
- 提示如何重新打开设置：在对话窗输入 `/settings`
- 点击"完成"后：
  1. 窗口关闭
  2. 程序进入隐身模式
  3. 延迟2秒自动弹出一次对话窗，AI发送欢迎消息确认一切正常

## 5. 交互模式

### 5.1 模式A：悬浮对话窗

主要交互方式。

**唤出方式：** 按用户设定的热键

**窗口特性：**
- 无边框、半透明背景、圆角
- `WS_EX_TOOLWINDOW`：不出现在任务栏和Alt+Tab
- 窗口创建后设置 `SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)`：截屏/录屏不可见
- 尺寸约 420x520（可拖拽调整），弹出位置根据用户设置
- 弹出时 100-150ms 淡入动画
- 消失时无动画直接销毁（为老板键场景保证速度）

**窗口内部布局：**
```
┌─────────────────────────┐
│  ● 对话标题    [—] [×]  │  ← 迷你标题栏，可拖动窗口
│─────────────────────────│
│                         │
│  消息流区域              │  ← 对话历史，支持滚动
│  （用户消息 + AI回复）    │
│                         │
│─────────────────────────│
│  [📎] [📷] 输入框  [发送] │  ← 底部输入区
└─────────────────────────┘
```

**输入行为：**
- 窗口弹出后输入框自动聚焦
- `Enter` 发送，`Shift+Enter` 换行
- 输入框左侧有附件按钮（发送文件/图片给AI）和截图按钮

**关闭行为：**
- `Esc` 或再按唤起热键 → 关闭（销毁窗口）
- 点击窗口外部 → 可配置是否自动关闭
- 对话历史保留在内存中，下次唤出继续

### 5.2 模式B：剪贴板快捷问答

**流程：**
1. 用户在任意应用中选中文字
2. 按剪贴板热键
3. 程序读取剪贴板内容，自动构造prompt发送给AI
4. 回答以**系统通知**弹出（伪装来源，见5.4节）

**回答策略：**
- 如果回答短（≤200字）→ 直接通知显示
- 如果回答长 → 通知显示摘要 + "点击查看完整内容"，点击后打开对话窗

### 5.3 模式C：截图发送AI

**流程：**
1. 用户按截图热键
2. 屏幕变暗，进入区域截图模式（类似微信/QQ截图的交互）
3. 用户框选区域后松开鼠标
4. 截图自动发送给AI（附带一句预设的prompt，如"请简洁回答图中的内容"）
5. AI回答以系统通知弹出（伪装来源）

**截图交互细节：**
- 进入截图模式时全屏覆盖一层半透明遮罩
- 鼠标拖动画出选区，选区内显示原始画面（亮），选区外变暗
- 松开鼠标自动确认并发送，不需要额外点击按钮（速度优先）
- 按 `Esc` 取消截图
- 截图遮罩本身也要设置截屏保护（防止截图工具截到截图界面）

**AI回答策略（针对截图场景）：**
- 系统prompt强制要求AI以极简方式回答
- 如果图中是选择题 → 回答格式："选B"或"B. xxx"
- 如果图中是填空题 → 直接给答案
- 如果图中是问答题 → 给最简要的要点
- 如果图中是代码/报错 → 给解决方案的关键步骤
- 如果图中是其他内容 → 一句话概括
- 用户可在设置中修改截图场景的系统prompt

### 5.4 通知伪装

系统通知可伪装为来自以下来源（用户在设置中选择）：

| 伪装来源 | 通知标题 | 图标 |
|---------|---------|------|
| QQ | QQ消息 | QQ图标 |
| 微信 | 微信 | 微信图标 |
| 浏览器 | Microsoft Edge | Edge图标 |
| 无伪装 | 空 | 无图标 |

- 通知内容就是AI的回答文字
- 点击通知 → 打开对话窗，显示完整对话上下文
- 如果AI回答太长，通知只显示前几行 + "..."

> 伪装图标需要内嵌对应的ico资源。图标不必完全一致，做到"扫一眼看不出区别"即可。

### 5.5 老板键

按下老板键后**立刻**执行：
1. 销毁当前对话窗（如果存在）
2. 关闭任何正在显示的通知
3. 取消截图模式（如果正在截图）
4. 不播放动画、不发出声音

恢复方式：再按唤起对话窗的热键。对话历史仍保留在内存中。

## 6. 对话系统

### 6.1 对话管理

- 支持多个对话session
- 对话窗内输入 `/new` 开启新对话
- 输入 `/list` 显示历史对话列表（按时间排序，显示每个对话的第一句话作为标题）
- 输入 `/clear` 清除当前对话历史
- 输入 `/delete` 删除当前对话记录（从本地数据库移除）

### 6.2 快捷指令

在对话窗输入框中，以 `/` 开头的指令为系统命令：

| 指令 | 功能 |
|------|------|
| `/settings` | 打开设置面板 |
| `/new` | 新建对话 |
| `/list` | 对话列表 |
| `/clear` | 清除当前对话 |
| `/delete` | 删除当前对话记录 |
| `/model` | 切换AI模型 |
| `/t <文字>` | 翻译（省略文字则翻译剪贴板） |
| `/s` | 总结剪贴板内容 |
| `/export` | 导出当前对话为文件 |

> 快捷指令列表可扩展，建议做成配置化的，方便后续添加。

### 6.3 AI回答风格控制

针对不同场景，发送给AI的system prompt不同：

- **普通对话模式**：正常对话，回答可以稍长但不啰嗦
- **截图模式**：强制简洁，直给答案，不解释过程
- **剪贴板模式**：根据内容自适应，翻译/总结/回答
- **用户可自定义system prompt**：在设置中修改各模式的提示词

### 6.4 本地存储

- 所有对话记录加密存储在本地SQLite数据库（`cache.db`）
- 加密方式：使用Windows DPAPI，绑定当前用户
- 存储内容：对话ID、时间戳、消息内容、所用模型
- 截图图片不默认保存，发送后从内存清除。可在设置中开启"保留截图历史"

## 7. 防检测系统

### 7.1 截屏保护

- 所有自身窗口（对话窗、设置面板、截图选区UI）创建后立即调用 `SetWindowDisplayAffinity(hwnd, WDA_EXCLUDEFROMCAPTURE)`
- 效果：截屏、录屏、远程桌面均无法捕获窗口内容
- 用户可在设置中关闭此功能

### 7.2 环境感知

程序持续监测运行环境，检测到以下情况时自动进入**静默模式**（暂停所有弹出行为）：

- 远程桌面连接处于活动状态（检测 `mstsc.exe`、`TeamViewer`、`AnyDesk` 等）
- 录屏软件正在运行（检测 `obs64.exe`、`bandicam.exe` 等）
- 会议软件正在共享屏幕（检测 `Zoom`、`Teams`、`腾讯会议` 的屏幕共享状态）
- 企业监控软件正在运行（可配置进程名列表）

> 检测进程名列表应做成可配置的，用户可以在设置中添加/移除。

### 7.3 任务管理器感知

检测到 `taskmgr.exe`（任务管理器）启动时：
- 暂停所有非必要的网络请求
- 降低内存占用
- 不做任何UI弹出

## 8. 设置面板

通过对话窗输入 `/settings` 或专用热键打开。

设置面板本身也是隐藏窗口（`WS_EX_TOOLWINDOW` + 截屏保护），标题伪装为"Display Adapter Configuration"。

### 设置分组：

**连接设置**
- AI服务商选择与API Key管理（支持多个）
- 默认使用的模型
- 自定义API endpoint
- 网络代理配置

**快捷键设置**
- 所有热键的重新绑定
- 冲突检测

**外观设置**
- 系统托盘图标开关
- 对话窗位置、大小、透明度
- 通知伪装来源选择
- 深色/浅色主题

**隐私与安全**
- 截屏保护开关
- 对话记录保留天数（默认永久，可设为自动清理）
- 截图历史保留开关
- 退出时清除所有数据开关

**行为设置**
- 开机自启动
- 点击窗口外部是否自动关闭对话窗
- 截图后自动发送 vs 手动确认
- 环境感知（静默模式）的进程名列表配置
- 各场景的system prompt自定义

**关于**
- 版本号
- 检查更新
- 完全卸载（清除所有文件、计划任务、注册表项、数据）

## 10. 技术实现要点

### 10.1 技术栈

- 语言：Python
- GUI框架：PyQt6（对话窗、设置面板、截图覆盖层）
- 打包：PyInstaller（`--onefile --noconsole`）
- 本地存储：SQLite + DPAPI加密
- 截图：PyQt6截屏 或 `mss` 库
- 热键：`ctypes` 调用 `RegisterHotKey`（无需管理员权限）
- 通知：Windows Toast Notification API（`win10toast` 或 `plyer` 或直接调用COM接口）

### 10.2 架构

```
主线程（Qt事件循环）
├── 全局热键监听（独立线程，Win32消息循环）
├── 对话窗管理器（按需创建/销毁QWidget）
├── 截图模块（全屏覆盖层QWidget）
├── 通知模块（系统Toast通知）
├── 环境感知模块（定时检测进程列表）
├── API通信模块（异步HTTP，独立线程/协程）
└── 数据持久化模块（加密SQLite读写）
```

### 10.3 窗口隐藏关键实现

所有自有窗口需要的WindowFlags组合：
- `Qt.FramelessWindowHint`（无边框）
- `Qt.WindowStaysOnTopHint`（置顶）
- `Qt.Tool`（不出现在任务栏）
- `WA_TranslucentBackground`（透明背景）
- Win32附加：`WS_EX_TOOLWINDOW | WS_EX_NOACTIVATE`

### 10.4 打包配置

- 输出文件名：`DAHService.exe`
- 无控制台窗口
- 内嵌图标资源（伪装通知用的QQ/微信/Edge图标）
- PE文件VersionInfo设置为系统服务风格的描述
